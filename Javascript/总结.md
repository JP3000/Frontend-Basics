# JavaScript 总结（对象、引用、浅拷贝）

## 1. 对象是什么

- 对象是键值对的集合，例如 `{ name: 'Tom', age: 18 }`。
- 对象属于引用类型：变量里存的是“引用（地址）”，不是对象本体。

## 2. 为什么两个看起来一样的对象不全等

```js
console.log({ a: 1 } === { a: 1 }); // false
```

- `===` 比较对象时，比较的是引用是否相同。
- 上面是两个新对象，引用不同，所以是 `false`。

## 3. 引用共享

```js
const obj1 = { a: 1 };
const obj2 = obj1;

obj2.a = 2;
console.log(obj1.a); // 2
```

- `obj2 = obj1` 后，两个变量指向同一个对象。
- 改 `obj2` 会影响 `obj1`。

## 4. 浅拷贝的核心

- 浅拷贝只复制第一层。
- 如果第一层属性是对象/数组（引用类型），复制的是引用，不是深层内容。

```js
const objA = { a: 1, nested: { b: 2 } };
const objB = { ...objA }; // 浅拷贝

console.log(objA === objB); // false（外层新对象）
console.log(objA.nested === objB.nested); // true（深层引用共享）
```

## 5. 面试一句话

浅拷贝会创建新的外层对象，但内部嵌套对象仍复用原引用，所以修改深层数据会互相影响。
