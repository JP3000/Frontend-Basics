## 第一题：累乘个位数

**题目描述**  
给定一个长度为 n 的整数数组 a₁, a₂, …, aₙ。  
定义 s₁ = a₁  
s₂ = a₁ × a₂  
s₃ = a₁ × a₂ × a₃  
…  
sₙ = a₁ × a₂ × … × aₙ  

请依次输出 s₁, s₂, …, sₙ 的个位数。

**输入格式**  
第一行：一个整数 n（1 ≤ n ≤ 10⁶）  
第二行：n 个整数 aᵢ（1 ≤ aᵢ ≤ 10⁹）

**输出格式**  
一行，n 个整数，每个数为对应 sᵢ 的个位数，用空格分隔。

**样例**  
输入：
```
4
2 3 5 7
```
输出：
```
2 6 0 0
```
**样例解释**  
s₁ = 2，个位 2  
s₂ = 6，个位 6  
s₃ = 30，个位 0  
s₄ = 210，个位 0

## 第二题：分解为最大素数

**题目描述**  
给定正整数 n、k、m，要求将 n 分解为恰好 k 个素数的和（素数可以重复）。  
这 k 个素数中，奇素数的个数必须是 m 的倍数（即个数为 m, 2m, 3m, …，且不能为 0）。  
在所有满足条件的分解中，找出字典序最大的分解（即按非递增顺序排列后，使得序列尽可能大）。  
若不存在这样的分解，输出 -1。

**输入格式**  
一行三个整数：n, k, m  
（数据范围：1 ≤ n ≤ 10⁹，1 ≤ k ≤ 10⁵，1 ≤ m ≤ 10⁵）

**输出格式**  
若存在，输出 k 个素数，按非递增顺序排列，用空格分隔。  
若不存在，输出 -1。

**样例**  
输入：
```
20 4 2
```
输出：
```
7 5 3 5
```
**样例解释**  
奇素数为 3 和 5，共 2 个，是 m=2 的倍数。  
总和 7+5+3+5=20。  
其他分解如 7+7+3+3=20，但奇素数 3,3 共 2 个，也是 2 的倍数，但序列 [7,7,3,3] 字典序小于 [7,5,3,5]（因为第二位 5 < 7，所以 [7,5,...] 更大？实际上比较时是第一位相同，第二位 5 vs 7，7 更大，所以 [7,7,3,3] 更大。因此需要明确字典序比较规则——通常是从大到小排列后，第一位尽量大，第二位尽量大，等等。所以 [7,7,3,3] 优于 [7,5,3,5]。）

## 第三题：最大划分次数

**题目描述**  
给定一个长度为 n 的正整数数组 a₁, a₂, …, aₙ。  
你可以进行如下操作：  
选择当前数组中的某一个连续子数组（初始时选择整个数组），如果该子数组的和是偶数，则可以将其从中间某个位置切分成两个连续的子数组，要求这两个子数组的和相等。  
切完后，这两个子数组成为独立的子数组，可以继续对它们进行同样的操作（只要满足和是偶数且能切出两个和相等的部分）。  
问最多能进行多少次划分操作。

**输入格式**  
第一行：一个整数 n（1 ≤ n ≤ 10⁵）  
第二行：n 个正整数 aᵢ（1 ≤ aᵢ ≤ 10⁴）

**输出格式**  
一个整数，表示最大操作次数。

**样例**  
输入：
```
4
1 2 2 1
```
输出：
```
1
```
**样例解释**  
整个数组和 = 6，可以切分成 [1,2]（和=3）和 [2,1]（和=3），操作 1 次。  
之后 [1,2] 和=3 奇数，无法再切；[2,1] 同理。  
所以总次数为 1。

## 第四题：子数组统计（原题缺失，以经典题代替）

**题目描述**  
给定一个长度为 n 的整数数组 a₁, a₂, …, aₙ 和一个整数 k。  
请统计有多少个连续子数组的和等于 k。

**输入格式**  
第一行：两个整数 n, k（1 ≤ n ≤ 2×10⁵，-10⁹ ≤ k ≤ 10⁹）  
第二行：n 个整数 aᵢ（-10⁴ ≤ aᵢ ≤ 10⁴）

**输出格式**  
一个整数，表示和为 k 的子数组个数。

**样例**  
输入：
```
5 3
1 1 1 1 1
```
输出：
```
3
```
**样例解释**  
子数组 [1,1,1]（下标 0-2）、[1,1,1]（下标 1-3）、[1,1,1]（下标 2-4），共 3 个。
