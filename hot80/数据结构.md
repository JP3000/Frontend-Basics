## 1. 数组转树
### 推导链
Q1: 输入什么？→ 扁平数组 `[{id, pid, name}, ...]`  
Q2: 输出什么？→ 树结构 `[{id, name, children: [...]}]`  
Q3: 怎么找父子关系？→ 通过 pid 找 parent  
Q4: 递归版问题？→ 每个节点都要遍历整个数组找子节点，O(n²)  
Q5: 怎么优化？→ Map 存所有节点，一次遍历建立关系，O(n)

### 变体速查
| 形态 | 核心 | 复杂度 |
| --- | --- | --- |
| 递归版 | filter 找子节点，递归处理 | O(n²) |
| Map 版 | 先建 Map，再遍历连接父子 | O(n) |


### 代码
```javascript
// 测试数据
const arrayData = [
  { id: 1, name: '部门A', pid: 0 },
  { id: 2, name: '部门B', pid: 1 },
  { id: 3, name: '部门C', pid: 1 },
  { id: 4, name: '部门D', pid: 2 },
  { id: 5, name: '部门E', pid: 2 },
  { id: 6, name: '部门F', pid: 3 },
]

// 写法一：递归版
function arrayToTree(arr, rootId = 0) {
    return  arr.filter(v => v.pid === rootId)
    .map(v => {
        const children = arrayToTree(arr,v.id)
        return children.length ? {...v, children} : {...v}
    })
}

// 写法二：Map 版（O(n)）
function arrayToTreeMap(arr, rootId = 0) { 
    const root = []
    const nodesMap = new Map()

    for(const {id, pid, ...args} of arr) {
      nodesMap.set(id, {
         id,
         pid,
         ...args,
         children: []
      })
    }

    for (const item of arr) {
      const node = nodesMap.get(item.id)

      if (node.pid === rootId) {
        root.push(node)
      } else {
        nodesMap.get(node.pid).children.push(node)
      }
    }

    return root
}

// 测试
console.log(JSON.stringify(arrayToTree(arrayData), null, 2))
console.log(JSON.stringify(arrayToTreeMap(arrayData), null, 2))
```

## 2. 树转数组
### 推导链
Q1: 输入什么？→ 树结构  
Q2: 输出什么？→ 扁平数组，带 pid  
Q3: 怎么遍历？→ DFS，递归处理 children  
Q4: pid 怎么来？→ 递归时传入父节点 id

### 代码
```javascript
function treeToArray(tree, pid = 0) {
  const result = []
  const nodes = Array.isArray(tree) ? tree : [tree]
  
  function dfs(nodes, parentId) {
    for (const node of nodes) {
      const { children, ...rest } = node
      result.push({ ...rest, pid: parentId })
      if (children?.length) {
        dfs(children, node.id)
      }
    }
  }
  
  dfs(nodes, pid)
  return result
}
```

## 3. 路径字符串转树
### 推导链
Q1: 输入什么？→ `['/a/b/c', '/a/b/d', '/a/e']`  
Q2: 输出什么？→ 树结构，相同前缀合并  
Q3: 怎么处理？→ 按 `/` 分割，逐层创建节点  
Q4: 怎么避免重复创建？→ Map 存 path -> node 映射

### 代码									
```javascript
function pathsToTree(paths) {
  const root = { name: '', children: [] }
  const map = new Map()
  map.set('', root)
  
  for (const path of paths) {
    const parts = path.split('/').filter(Boolean)
    let curPath = ''
    
    for (const part of parts) {
      const parentPath = curPath
      curPath = curPath + '/' + part
      
      if (!map.has(curPath)) {
        const node = { name: part, children: [] }
        map.set(curPath, node)
        map.get(parentPath).children.push(node)
      }
    }
  }
  
  return root
}

// 测试
const paths = ['/a/b/c', '/a/b/d', '/a/e/f', '/g']
console.log(JSON.stringify(pathsToTree(paths), null, 2))
```

## 4. 按缩进构造树
### 推导链
Q1: 输入什么？→ 带缩进的文本  
Q2: 怎么判断层级？→ 缩进空格数 / 缩进单位  
Q3: 怎么找父节点？→ Map 存每个 level 的最后一个节点  
Q4: 缩进减少时？→ 清理更深层级的 Map 记录

### 代码
```javascript
  function buildTreeByIndent(text, indentSize = 2) {
    const lines = text.split('\n').filter(line => line.trim())
    const levelMap = new Map()
    const root = []
    
    for (const line of lines) {
      const indent = line.search(/\S/)  // 第一个非空格位置
      const level = Math.floor(indent / indentSize)
      const value = line.trim()
      const node = { value, children: [] }
      
      if (level === 0) {
        root.push(node)
      } else {
        levelMap.get(level - 1)?.children.push(node)
      }
      
      levelMap.set(level, node)
      
      // 清理更深层级
      for (let i = level + 1; i <= levelMap.size; i++) {
        levelMap.delete(i)
      }
    }
    
    return root
  }
  
  // 测试
  const text = `A
    B
        C
    D
            E`
// 如果不清理更深层级就会导致E挂在C上面
  console.log(JSON.stringify(buildTreeByIndent(text), null, 2))
```

## 5. 二叉树遍历
### 推导链
Q1: 三种顺序？  
→ 前序：根 → 左 → 右  
→ 中序：左 → 根 → 右  
→ 后序：左 → 右 → 根

Q2: 递归怎么写？→ 改变 push 位置  
Q3: 迭代怎么写？→ 用栈模拟递归  
Q4: 后序迭代技巧？→ 根右左 然后 reverse

### 变体速查
| 遍历 | 递归顺序 | 栈迭代技巧 |
| --- | --- | --- |
| 前序 | 根左右 | 先 push 右再 push 左 |
| 中序 | 左根右 | 一路向左入栈，弹出处理，转右 |
| 后序 | 左右根 | 根右左 + reverse |


### 代码
```javascript
class TreeNode {
  constructor(val, left = null, right = null) {
    this.val = val
    this.left = left
    this.right = right
  }
}

// 构建测试树:  1
//            / \
//           2   3
//          / \
//         4   5
const root = new TreeNode(1,
  new TreeNode(2, new TreeNode(4), new TreeNode(5)),
  new TreeNode(3)
)

// 递归版
function preorderRecur(root, res = []) {
  if (!root) return res
  res.push(root.val)
  preorderRecur(root.left, res)
  preorderRecur(root.right, res)
  return res
}

function inorderRecur(root, res = []) {
  if (!root) return res
  inorderRecur(root.left, res)
  res.push(root.val)
  inorderRecur(root.right, res)
  return res
}

function postorderRecur(root, res = []) {
  if (!root) return res
  postorderRecur(root.left, res)
  postorderRecur(root.right, res)
  res.push(root.val)
  return res
}

// 迭代版
function preorder(root) {
  if (!root) return []
  const res = [], stack = [root]
  while (stack.length) {
    const node = stack.pop()
    res.push(node.val)
    if (node.right) stack.push(node.right)  // 先右后左，出栈就是先左后右
    if (node.left) stack.push(node.left)
  }
  return res
}

function inorder(root) {
  const res = [], stack = []
  let cur = root
  while (cur || stack.length) {
    while (cur) {  // 一路向左
      stack.push(cur)
      cur = cur.left
    }
    cur = stack.pop()
    res.push(cur.val)
    cur = cur.right
  }
  return res
}

function postorder(root) {
  if (!root) return []
  const res = [], stack = [root]
  while (stack.length) {
    const node = stack.pop()
    res.push(node.val)
    if (node.left) stack.push(node.left)   // 先左后右
    if (node.right) stack.push(node.right)
  }
  return res.reverse()  // 根右左 → 左右根
}

// 测试
console.log('前序:', preorder(root))   // [1,2,4,5,3]
console.log('中序:', inorder(root))    // [4,2,5,1,3]
console.log('后序:', postorder(root))  // [4,5,2,3,1]
```

## 7. LRU 缓存
### 推导链
Q1: LRU 是什么？→ 最近最少使用，满了淘汰最久没用的  
Q2: 需要什么操作？→ get O(1)、put O(1)  
Q3: 怎么实现 O(1)？  
→ Map 版：利用 Map 的插入顺序，删除再插入就到末尾  
→ 双向链表版：头部最新，尾部最旧

### 变体速查
| 形态 | 核心 | 适用 |
| --- | --- | --- |
| Map 版 | delete + set 移到末尾 | 面试首选，简单 |
| 双向链表版 | 头插 + 尾删 | 考察数据结构 |


### 代码
```javascript
// 写法一：Map 版（推荐）
class LRUCache {
  constructor(capacity) {
    this.capacity = capacity
    this.cache = new Map()
  }
  
  get(key) {
    if (!this.cache.has(key)) return -1
    const value = this.cache.get(key)
    // 删除再插入，移到末尾（最新）
    this.cache.delete(key)
    this.cache.set(key, value)
    return value
  }
  
  put(key, value) {
    if (this.cache.has(key)) {
      this.cache.delete(key)
    } else if (this.cache.size >= this.capacity) {
      // 删除第一个（最旧）
      const firstKey = this.cache.keys().next().value
      this.cache.delete(firstKey)
    }
    this.cache.set(key, value)
  }
}

// 写法二：双向链表版
class LRUCacheLinked {
  constructor(capacity) {
    this.capacity = capacity
    this.cache = new Map()
    // 虚拟头尾节点
    this.head = { key: 0, val: 0, prev: null, next: null }
    this.tail = { key: 0, val: 0, prev: null, next: null }
    this.head.next = this.tail
    this.tail.prev = this.head
  }
  
  get(key) {
    if (!this.cache.has(key)) return -1
    const node = this.cache.get(key)
    this.moveToHead(node)
    return node.val
  }
  
  put(key, value) {
    if (this.cache.has(key)) {
      const node = this.cache.get(key)
      node.val = value
      this.moveToHead(node)
    } else {
      const node = { key, val: value, prev: null, next: null }
      this.cache.set(key, node)
      this.addToHead(node)
      if (this.cache.size > this.capacity) {
        const removed = this.removeTail()
        this.cache.delete(removed.key)
      }
    }
  }
  
  addToHead(node) {
    node.prev = this.head
    node.next = this.head.next
    this.head.next.prev = node
    this.head.next = node
  }
  
  removeNode(node) {
    node.prev.next = node.next
    node.next.prev = node.prev
  }
  
  moveToHead(node) {
    this.removeNode(node)
    this.addToHead(node)
  }
  
  removeTail() {
    const node = this.tail.prev
    this.removeNode(node)
    return node
  }
}

// 测试
const lru = new LRUCache(2)
lru.put(1, 'a')
lru.put(2, 'b')
console.log(lru.get(1))  // 'a'
lru.put(3, 'c')          // 淘汰 key 2
console.log(lru.get(2))  // -1
```

## 8. 四大排序
### 推导链
Q1: 快排思路？→ 选 pivot，小的放左边大的放右边，递归  
Q2: 归并思路？→ 分两半，递归排序，合并有序数组  
Q3: 冒泡思路？→ 相邻比较，大的往后冒  
Q4: 插入思路？→ 像打牌，把每张牌插入已排序部分

### 变体速查
| 排序 | 时间复杂度 | 空间 | 稳定 |
| --- | --- | --- | --- |
| 快排 | O(n log n) | O(log n) | ❌ |
| 归并 | O(n log n) | O(n) | ✅ |
| 冒泡 | O(n²) | O(1) | ✅ |
| 插入 | O(n²) | O(1) | ✅ |


### 代码
```javascript
  // 快排（简单版）
// 心智模型：选一个 pivot，把剩下元素分成“小于它”和“大于等于它”两堆。
// 写法顺序：终止条件 -> 分 left/right -> 排左 + pivot + 排右。
// 注意：这版靠新数组换可读性，不是原地；重复值放哪边都行，但规则要一致。
function quickSortA(arr) {
  if (arr.length <= 1) return arr
  const pivot = arr[0]
  const left = [], right = []
  for (let i = 1; i < arr.length; i++) {
    arr[i] < pivot ? left.push(arr[i]) : right.push(arr[i])
  }
  return [...quickSortA(left), pivot, ...quickSortA(right)]
}

// 快排（原地版）
// 心智模型：lo 从左找“该去右边的”，hi 从右找“该去左边的”，找到就交换。
// 分区结束时：hi 是左半边尾，lo 是右半边头，所以递归写 [l, hi] 和 [hi + 1, r]。
// 注意：不能只把 hi 机械换成 lo 还沿用同一递归区间，否则区间可能不缩小。
function quickSortB(arr, l = 0, r = arr.length - 1) {
    if (l >= r) return arr
    const pivot = arr[l]
    let lo = l - 1, hi = r + 1

    while (lo < hi) {
        do lo++; while (arr[lo] < pivot)
        do hi--; while (arr[hi] > pivot)
        if (lo >= hi) break
        [arr[lo], arr[hi]] = [arr[hi], arr[lo]]
    }

    quickSortB(arr, l, hi)
    quickSortB(arr, hi+1, r)
    return arr
}

// 归并排序
// 心智模型：先拆到每段只剩 1 个数，再把两个有序段像拉链一样合回去。
// 写法顺序：切 mid -> 排左 -> 排右 -> merge 里双指针谁小拿谁。
// 注意：merge 想稳定，值相等时先拿左边；while 条件右侧要看 right.length。
function mergeSort(arr) {
    if (arr.length <= 1) return arr
    const mid = arr.length >> 1
    const l = mergeSort(arr.slice(0, mid))
    const r = mergeSort(arr.slice(mid))
    return merge(l, r)
}

// merge：双指针谁小拿谁，一边用完后，把另一边剩余部分整段接上。
function merge(left, right) {
    let i = 0, j = 0
    const res = []
    while (i < left.length && j < left.length) {
        res.push(left[i] < right[j] ? left[i++] : right[j++])
    }
    return res.concat(left.slice(i), right.slice(j))
}

// 冒泡排序
// 心智模型：每一轮只做一件事，把当前最大值一路“冒”到末尾。
// 写法顺序：外层控制轮数，内层比较相邻元素，谁大谁往后换，没交换就提前停。
// 注意：内层上界是 len - 1 - i；只在 arr[j] > arr[j + 1] 时交换。
// 过程例子：[5, 3, 8, 1]
// 第 1 轮后：[3, 5, 1, 8]，最大值 8 到末尾。
// 第 2 轮后：[3, 1, 5, 8]，次大值 5 到倒数第二。
// 第 3 轮后：[1, 3, 5, 8]，完成。
function bubbleSort(arr) {
    const len = arr.length
    for (let i=0; i<len-1; i++) {
        let flag = false
        for (let j=0; j<len-1-i; j++) {
            if (arr[j] > arr[j+1]) {
                [arr[j], arr[j+1]] = [arr[j+1], arr[j]]
                flag = true
            }   
        }

        if (!flag) break
    }
    return arr
}


// 插入排序
// 心智模型：左边永远有序，把当前牌 cur 抽出来，往左找坑，再塞回去。
// 写法顺序：先存 cur，再把比 cur 大的元素整体右移，最后把 cur 放到 j + 1。
// 注意：一定先保存 cur；比较写 arr[j] > cur 时，插入排序才稳定。
// 过程例子：[5, 3, 8, 1]
// i = 1：把 3 插到 5 前面 -> [3, 5, 8, 1]
// i = 2：8 已经比前面大，不动 -> [3, 5, 8, 1]
// i = 3：把 1 插到最前面 -> [1, 3, 5, 8]
function insertionSort(arr) {
  for (let i=1; i<arr.length; i++) {
    let cur = arr[i]
    let j = i-1
    while (j >= 0 && arr[j] > cur) {
        arr[j+1] = arr[j]
        j--
    }
    arr[j+1] = cur
  }
  return arr
}


// 测试
const arr = [5, 2, 8, 1, 9, 3]
console.log('快排:', quickSortA([...arr]))
console.log('快排', quickSortB([...arr], 0, arr.length-1))
console.log('归并:', mergeSort([...arr]))
console.log('冒泡:', bubbleSort([...arr]))
console.log('插入:', insertionSort([...arr]))

```

## 9. 课程表（图的环检测）
### 推导链
Q1: 问题是什么？→ 判断课程能否全部修完（有没有循环依赖）  
Q2: 本质是什么？→ 有向图检测环  
Q3: 怎么建图？→ 邻接表 `graph[a] = [b, c]` 表示 a 依赖 b、c  
Q4: 怎么检测环？  
→ DFS：三色标记（白/灰/黑），遇到灰色说明有环  
→ BFS：拓扑排序，入度为 0 的先处理，最后看是否全处理完

### 变体速查
| 方法 | 核心 | 判断有环 |
| --- | --- | --- |
| DFS 三色 | 白→灰→黑，灰色遇灰色 | 访问到灰色节点 |
| BFS 拓扑 | 入度为 0 入队，处理后减入度 | 处理数 < 总数 |


### 代码
```javascript
// 输入：numCourses = 4, prerequisites = [[1,0],[2,0],[3,1],[3,2]]
// 表示：修 1 要先修 0，修 2 要先修 0，修 3 要先修 1 和 2

// 写法一：DFS 三色标记
function canFinishDFS(numCourses, prerequisites) {
  // 建图：graph[i] = [依赖的课程]
  const graph = Array.from({ length: numCourses }, () => [])
  for (const [cur, pre] of prerequisites) {
    graph[cur].push(pre)
  }
  
  // 0: 未访问, 1: 访问中, 2: 已完成
  const visited = new Array(numCourses).fill(0)
  
  function dfs(node) {
    if (visited[node] === 1) return false  // 遇到灰色，有环
    if (visited[node] === 2) return true   // 已处理过
    
    visited[node] = 1  // 标记为访问中
    for (const next of graph[node]) {
      if (!dfs(next)) return false
    }
    visited[node] = 2  // 标记为已完成
    return true
  }
  
  for (let i = 0; i < numCourses; i++) {
    if (!dfs(i)) return false
  }
  return true
}

// 写法二：BFS 拓扑排序
function canFinishBFS(numCourses, prerequisites) {
  const graph = Array.from({ length: numCourses }, () => [])
  const inDegree = new Array(numCourses).fill(0)
  
  // 建图 + 统计入度
  for (const [cur, pre] of prerequisites) {
    graph[pre].push(cur)  // pre → cur
    inDegree[cur]++
  }
  
  // 入度为 0 的入队
  const queue = []
  for (let i = 0; i < numCourses; i++) {
    if (inDegree[i] === 0) queue.push(i)
  }
  
  let count = 0
  while (queue.length) {
    const node = queue.shift()
    count++
    for (const next of graph[node]) {
      inDegree[next]--
      if (inDegree[next] === 0) queue.push(next)
    }
  }
  
  return count === numCourses
}

// 测试
console.log(canFinishDFS(4, [[1,0],[2,0],[3,1],[3,2]]))  // true
console.log(canFinishDFS(2, [[1,0],[0,1]]))              // false（有环）
console.log(canFinishBFS(4, [[1,0],[2,0],[3,1],[3,2]]))  // true
console.log(canFinishBFS(2, [[1,0],[0,1]]))              // false
```

## 10. 路径总和
### 推导链
Q1: 路径总和 I？→ 判断是否存在根到叶路径和等于 target  
Q2: 路径总和 II？→ 返回所有符合的路径  
Q3: 路径总和 III？→ 路径不一定从根开始，不一定到叶结束  
Q4: 核心技巧？→ DFS + 回溯，传递剩余目标值

### 代码
```javascript
class TreeNode {
  constructor(val, left = null, right = null) {
    this.val = val
    this.left = left
    this.right = right
  }
}

// 路径总和 I：是否存在
function hasPathSum(root, target) {
  if (!root) return false
  // 叶子节点：判断剩余值是否等于当前节点值
  if (!root.left && !root.right) return root.val === target
  const remain = target - root.val
  return hasPathSum(root.left, remain) || hasPathSum(root.right, remain)
}

// 路径总和 II：返回所有路径
function pathSum(root, target) {
  const result = []
  
  function dfs(node, remain, path) {
    if (!node) return
    path.push(node.val)
    
    // 叶子节点且路径和等于目标
    if (!node.left && !node.right && remain === node.val) {
      result.push([...path])
    }
    
    dfs(node.left, remain - node.val, path)
    dfs(node.right, remain - node.val, path)
    path.pop()  // 回溯
  }
  
  dfs(root, target, [])
  return result
}

// 路径总和 III：任意起点终点
function pathSumIII(root, target) {
  if (!root) return 0
  
  // 从当前节点开始的路径数
  function countFromNode(node, remain) {
    if (!node) return 0
    let count = node.val === remain ? 1 : 0
    count += countFromNode(node.left, remain - node.val)
    count += countFromNode(node.right, remain - node.val)
    return count
  }
  
  // 总数 = 从根开始 + 左子树的 + 右子树的
  return countFromNode(root, target) + 
         pathSumIII(root.left, target) + 
         pathSumIII(root.right, target)
}

// 构建测试树:      5
//              /     \
//             4       8
//            /       / \
//           11      13  4
//          / \        / \
//         7   2      5   1
const root = new TreeNode(5,
  new TreeNode(4, new TreeNode(11, new TreeNode(7), new TreeNode(2))),
  new TreeNode(8, new TreeNode(13), new TreeNode(4, new TreeNode(5), new TreeNode(1)))
)

console.log(hasPathSum(root, 22))  // true (5→4→11→2)
console.log(pathSum(root, 22))     // [[5,4,11,2]]
console.log(pathSumIII(root, 8))   // 3 (5→3, 11→-3, 4→4)
```

## 11. 商品列表的diff （拼多多笔试)
实现一个，比较编辑前后商品列表，找出哪些商品是新增，哪些商品被去除，哪些字段被修改了。 (diff)

```javascript
const oldList = [
  { id: 1, name: '苹果', price: 5 },
  { id: 2, name: '香蕉', price: 3 }
]

const newList = [
  { id: 1, name: '苹果', price: 6 },
  { id: 3, name: '西瓜', price: 8 }
]
// {
//   add: [
//     { id: 3, name: '西瓜', price: 8 }
//   ],
//   remove: [
//     { id: 2, name: '香蕉', price: 3 }
//   ],
//   update: [
//     {
//       id: 1,
//       changes: {
//         price: {
//           before: 5,
//           after: 6
//         }
//       }
//     }
//   ]
// }

function diffGoods(oldList, newList) {
  const add = [], remove = [], update = []
  for (const newItem of newList) {
    // 新增模块
    const oldItem  = oldList.find(v => v.id === newItem.id)
    if (!oldItem) {
        add.push(newItem)
        continue
    }
    // 更改模块
    const changes = {}
    const keys = new Set([...Object.keys(oldItem), ...Object.keys(newItem)])
    for (const key in keys) {
        if (key === 'id' || newItem[key] === oldItem[key]) continue
        changes[key] = {
            before: oldItem[key],
            after: newItem[key]
        }
    }
    update.push({id: newItem.id, changes})
  }

  for (const oldItem of oldList) {
    const newItem = newList.find(v => v.id === oldItem.id)
    if (!newItem) {
        remove.push(oldItem)
    }
  }

  return { add, remove, update }
}
```
